Cocos2d-x的动作

发表于 2016-08-02   |   分类于 Cocos2d-x基础知识系列   |   | 阅读量 次

　　通过前几篇博文我们知道了一些基本概念和Sprite、Layer与Scene的创建，我们学习了这些就可以创建自己的游戏场景界面等，但是，创建出来的场景界面是静态的，而游戏中主要是动态的[/手动惊恐]，各位小伙伴不要惊慌，Cocos2d-x中声明了一个Action（动作）类，下面船长就来说说Action类及其子类的概述用法等。

Action 动作类

　　动作类（Action）是所有动作的基类，它创建的一个对象代表一个动作。动作作用在Node上，因此每一个动作都需要由Node对象来执行。动作类（Action）作为基类，实际上是一个接口，动作类的大多数实现类都派生于有限时间动作类（FiniteTimeAction）。当然，Action类的子类还有Follow与Speed两个动作类。Action的子类及其父类的关系图解如下：

FiniteTimeAction 有限时间动作类

　　FiniteTimeAction类继承自Action类，其子类有ActionInterval（延时动作）类与ActionInstant（瞬时动作）类，在实际游戏开发中经常用到这两个子类中的子类动作。
