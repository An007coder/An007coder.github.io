Cocos2d-x之动画

　　上一篇博文讲了各种各样的动作（虽然船长只举例讲了几类动作，但是Cocos2d-x中封装了比较多的动作[/手动尴尬]），如果大家掌（练）握（习）地还算熟练的话，自己做出一些比较复（酷）杂（炫）的动作应该还是木有问题的吧。But，大家有木有觉得砸门用那些动作组合出来的复杂动作很僵硬，总感觉差点什么，有木有？？？下面船长就来讲讲解决这种僵硬的方法之动画，Cocos2d-x中的动画有两种，分别是序列帧动画和骨骼动画。这里船长就不bb了，下面直接进入正题。

序列帧动画

　　Cocos2d-x中动画的具体内容是依靠精灵显示出来的，为了显示动态图片，砸门需要不停切换精灵显示的内容，通过把静态的精灵变为动画播放器从而实现动画效果。序列帧动画由帧组成，每一帧都是一个纹理。使用Animation类描述一个动画，而精灵显示动画的动作则是一个Animate对象。动画动作Animate是精灵显示动画的动作，它由一个动画对象创建，并由精灵执行。直接上代码，如下：

　　一般情况下，对于一个精灵动画每次创建时都要通过上述繁琐过程。当一个动作使用频率较高时，砸门将其加入到AnimationCache（动画缓存）中可以提高游戏的性能。AnimationCache类接口如下：
　　♠ static AnimationCache＊ getInstance()：获得AnimationCache单例。
　　♠ void addAnimation(Animation＊ animation, const std::string& name)：将animation动画添加到动画缓存中，并设置其名字为name。
　　♠ void addAnimationsWithFile(const std::string& plist)：将plist动画文件添加到动画缓存中。
　　♠ void removeAnimation(const std::string& name)：将名字为name的动画从动画缓存中移除。
　　♠ Animation＊ getAnimation(const std::string& name)：从动画缓存中获得名字为name的动画。
　　接口的使用很简单，这里船长就不说了。

骨骼动画

　　当游戏中的动作较多时，使用序列帧动画就会造成包体大等问题，那么怎么减小包体呢？此时就应该使用骨骼动画来代替序列帧动画，序列帧动画是由精灵动作中不同时间的快照组成的，快照数（帧数）的多少决定着动作的流畅性，而骨骼动画是将精灵拆分成多个小的部分（骨骼），然后改变部分骨骼的动作状态来改变精灵的动作状态。制作骨骼动画的方法有几种我不知道，但是不止两三种，每种导出的文件都不同，在Cocos2d-x中的使用方法也不同。
