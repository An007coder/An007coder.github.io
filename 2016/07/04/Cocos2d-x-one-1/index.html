Cocos2d-x的Node与Scene的概述

发表于 2016-07-04   |   分类于 Cocos2d-x基础知识系列   |   | 阅读量 次

　　目前大多数游戏公司用的开发引擎是Cocos2d-x和Unity3D。本系列为Cocos2d-x，后期将Cocos2d-x系列写完也会写一个Unity3D的系列教程。下面正式进入Cocos2d-x系列教程。

本博文主要内容

　　★ Node 节点类的功能与用法
　　★ Scene 场景类的功能与用法

Node 节点类

　　节点类（Node）是Cocos2d-x中最重要的类，继承自Ref类。
　　注：Cocos2d-x 3.0之前，大部分类都继承Object类，在3.0之后将Object类重命名为Ref类。

　　Cocos2d-x中所有需要绘制到屏幕的对象，或是自身包含要绘制到屏幕中的对象，都是继承自Node类，最常用的节点类包括场景（Scene）、层（Layer）、精灵（Sprite）等等，每一个节点只有一个父节点，但是可有由无数个子节点。

　　一个节点的主要功能是：
　　★可以包含其他节点
　　★可以通过定时器预定消息和使用回调函数
　　★可以执行动作

Node 类的成员变量

　　Node类一些常用的受保护成员变量如下：
　　▲ _rotationX：表示节点X轴的水平旋转角度，节点以锚点为中心顺时针旋转，单位角度。变量类型为float，默认值为0。
　　▲ _rotationY：表示节点Y轴的垂直旋转角度，其他属性同 _rotationX。
　　▲ _scaleX：表示节点X轴的缩放比例，节点以锚点为中心缩放该比例。变量类型是float，默认值为1.0，其值小于1.0时为缩小，大于1.0时为放大。
　　▲ _scaleY：表示节点Y轴的缩放比例，其他属性同 _scaleX。
　　▲ _scaleZ：表示节点Z轴的缩放比例，3D坐标（X、Y、Z），其他属性同 _scaleX。
　　▲ _position：表示节点在屏幕中的位置。v3.0及以前版本时变量类型是Point，以point（点）为单位，默认只是屏幕的左下角坐标（0,0)。从v3.0以后版本建议变量类型使用Vec2，当然使用Point亦可。Position设置的是节点在父节点中的坐标位置。
　　▲ _skewX：表示节点X轴的倾斜角度，节点以锚点为中心，平行X轴顺时针变形，单位为角度，默认值为0。
　　▲ _skewY：表示节点Y轴的倾斜角度，节点以锚点为中心，平行Y轴逆时针变形，单位为角度，默认值为0。
　　▲ _anchorPoint：表示节点的锚点，其变量类型是Vec2。锚点的x和y的取值通常是0~1之间的实数，表示锚点相对于节点长宽的位置。例如（0，0）表示把节点的左下角作为锚点，（0，1）表示把节点的左上角作为锚点，（1，0）表示把节点的右下角作为锚点，（1，1）表示把节点的右上角作为锚点。节点默认的锚点为（0.5，0.5），即节点中心。
　　▲ _contentSize：表示未经转换的节点内容大小，变量类型是Size，默认值为（0，0），即长与宽。无论节点如何缩放和旋转，其值始终不变。对于精灵（Sprite）其值是该精灵的纹理显示部分的大小，对于层（Layer）和场景（Scene）其值的大小就是屏幕的大小。
　　▲ _localZOrder：表示节点相对于其“兄弟”节点（拥有共同父节点）的Z顺序值，变量类型是int。对于像精灵（Sprite）这种有视觉呈现的节点，该参数决定了节点的绘制顺序，Z值越小越先绘制，相同Z值越先调用添加到父节点越先绘制，因此zOrder值大的节点会“遮挡”zOrder值小的节点。
　　▲ _children：节点的子节点数组，变量类型是Vector。
　　▲ _parent：节点的父节点，变量类型是Node。
　　▲ _tag：用于识别节点的标识值，变量类型是int。
　　▲ _visible：表示节点的是否可见，变量类型是bool，默认值为true，即可见，为false是不可见。
　　▲ _running：表示当前节点是否在运行，变量类型是bool。

Node 类的函数

　　Node类中的函数可以分为三大类，分别是对子节点的处理、如何执行动作和使用定时器预定消息，此外也有些函数不属于以上三大类，下面一一介绍。

处理子节点的函数

　　Node类实现了所有添加、获取和删除子节点的函数，下面是一些常用的处理子节点的函数。
△ virtual void addChild(Node* child)：将新节点添加为当前节点的子节点，zOrder值为0。

△ virtual void addChild(Node* child, int localZOrder)：将新节点添加为当前节点的子节点并设置子节点的zOrder值。

△ virtual void addChild(Node* child, int localZOrder, int tag)：将新节点添加为当前节点的子节点并设置子节点的zOrder值和tag值。

△ Node* getChildByTag(int m_tag)：在当前节点中获取tag值为m_tag的子节点，当子节点中有多个tag值为m_tag则返回第一个tag值为m_tag的子节点。

△ virtual Vector<Node*>& getChildren()：获取该节点的所有子节点。

△ virtual void removeChildByTag(int m_tag, bool cleanup = true)：通过tag值删除子节点。

△ virtual void removeChild(Node* child, bool cleanup = true)：通过节点指针删除子节点。

△ virtual void removeAllChildren()：删除该节点的所有子节点。

执行动作的函数

△ Action* runAction(Action* action)：运行某个特定的动作，并返回所执行的动作。

△ void stopAction(Action* action)：从运行列表中停止和删除一个特定的动作。

△ void stopActionByTag(int m_tag)：从运行的动作列表中停止并删除tag值为m_tag的动作。

△ void stopAllActions()：从运行的动作列表中停止并删除所有动作。

△ Action* getActionByTag(int m_tag)：从运行的动作列表中获取tag值为m_tag的动作。

定时器事件的相关函数

△ void scheduleUpdate(void)：启用定时器回调更新（update）函数，该函数每一帧都会被调用。每一个节点只可预定一个update函数，参数delta表示该函数上一次调用到这一次调用所经过的时间。Node默认并没有启用update回调更新函数，我们需要调用scheduleUpdate函数并重载update函数以执行自己的游戏逻辑。

△ void schedule(SEL_SCHEDULE selector, float interval)：如果需要根据不同的时间间隔来触发不同的定时更新逻辑，就需要指定运行特定的更新函数。schedule_selector是一个把指定函数转换成函数指针的宏，用于创建schedule函数所需的函数指针；传入这个宏的函数应该包含一个float参数，表示距离前一次触发事件的时间间隔。

△ void scheduleOnce(SEL_SCHEDULE selector, float delay)：指定运行特定的更新函数，并设置调用的延迟时间，但只运行一次。

△ void unscheduleUpdate(void)：停止该节点的每帧定时器。

△ void unschedule(SEL_SCHEDULE selector)：停止指定运行的自定义的定时器。

△ void unscheduleAllSelectors(void)：停止该节点的所有定时器。

其他函数

△ virtual bool init()：该函数用于初始化节点，返回bool值，表示是否初始化成功。
△ virtual void onEnter()：当节点所在场景即将呈现时调用，如果进入时带有过渡效果，则在过渡开始时调用。
△ virtual void onEnterTransitionDidFinish()：当节点所在场景入场过渡效果结束后调用，如果没有过渡效果，则紧接着在onEnter()后被调用。
△ virtual void onExit()：当节点所在场景即将退出时调用。
△ virtual void onExitTransitionDidStart()：当节点的场景的出场过渡效果结束后调用，如果没有过渡效果，则紧接着在onExit()后被调用。
△ virtual void cleanup()：该函数用于停止所有运行中的动作和定时器。
△ virtual void draw()：重写该函数以绘制自己的节点。

坐标系

　　无论是开发2D还是3D游戏，我们首先必须弄清楚坐标系的概念。在Cocos2d-x中，需要知道OpenGL坐标系、世界坐标系和节点坐标系等。

UI坐标系

　　OS／Android／Windows　SDK中的通用UI坐标系默认起点坐标（x=0，y=0）位于左上角，X轴坐标从屏幕最左边开始，由左向右渐增，即越右ｘ坐标越大；Y轴坐标从屏幕最上边开始，由上向下渐增，即越下ｙ坐标越大。
　　UI坐标系又称为纹理坐标系，在Cocos2d-x中，只有从纹理中截取部分矩形时才使用这个坐标系。

OpenGL坐标系

　　OpenGL坐标系又称为绘图坐标系，Cocos2d-x以OpenGL和OpenGL ES为基础，所以支持的自然是OpenGL坐标系。OpenGL坐标系默认原点坐标（x=0，y=0）在屏幕左下角，X轴坐标从屏幕的最左边开始，由左向右渐增；Y轴坐标从屏幕的最下边开始，由下向上渐增。

世界坐标系

　　世界坐标系又称为绝对坐标系，是游戏开发中的概念，它建立了描述其他坐标系所需要的参考标准，我们可以用世界坐标来描述其他坐标系的位置。
　　Cocos2d-x中的元素结构是有父子关系的层次结构，通过Node的position设置元素的位置使用的是相对其父节点的本地坐标系，而非世界坐标系。最后绘制在屏幕的时候，Cocos2d-x会把这些元素的本地节点坐标映射成世界坐标系坐标。世界坐标与OpenGL坐标系保持一致。

节点坐标系

　　节点坐标系又称为本地坐标系，是和特定节点相关联的坐标系，每个节点都有独立的坐标系。当节点移动或改变方向时，和该节点关联的坐标系（该节点的子节点）也将随之移动或改变方向，这一切都是相对的。

锚点

　　锚点是Cocos2d-x中一个很重要的概念。每一个Node都有一个锚点（anchor point），锚点指定了纹理图像和所在节点原点（即position所表示的点）重合的点的位置。锚点的设置在Node类的成员变量中有提过，这里就不再赘述了。

Scene 场景类

　　Scene类是继承Node类的，但只是简单的继承，并没有提供特别的功能。场景只是层的容器，包含了所有需要显示的游戏元素。除了作为层的容器，场景还起到了流程控制的作用，使用Director::getInstance()->replaceScene(newScene);等函数可以使游戏在不同的场景切换。
　　根据游戏的内容，可以将场景分为三大类：
　　★ 展示类场景：播放视频或输出文字，用来显示游戏的介绍、提示和帮助界面。
　　★ 选项类场景：游戏主菜单、设置参数等。
　　★ 游戏内容场景：显示游戏中的内容。
　　目前，Scene类基本上没有附加任何其他的逻辑，它的功能通常是根据像素坐标把元素放置在场景中的相应位置。因为Director（导演）类只能操作Scene类，所以实际开发中通常都是将Scene作为其他所有节点的父节点，看作Layer（层）对象的一个容器。而Scene的子节点通常都是继承直接继承Layer类，Layer包含各个游戏对象。因为大多数情况下场景对象本身不包含任何游戏相关的代码，而且很少被子类化，因此它一般在Layer对象里通过create静态函数来创建。

　　此外，如果要实现各种动画效果的场景切换，必须基于Scene场景进行切换。

TransitionScene 场景切换

　　在游戏中经常需要切换场景，在切换的时候需要一定的过渡效果，否则，场景的切换就会显得十分突兀。为此，Cocos2d-x提供了很多华丽的场景切换特效，例如淡入淡出、翻页、旋转等等，这些特效是通过TransitionScene系统特效类来实现的。
　　TransitionScene继承自Scene，而所有实现场景过渡效果的类都继承自TransitionScene。TransitionScene的主要函数如下：
　　★ void finish()：该函数在过渡效果完成后调用。
　　★ void hideOutShowIn()：部分过渡效果会使用该函数来隐藏更外面的场景。
　　★ bool initWithDuration(float time, Scene* scene)：该函数初始化一个场景过渡效果，参数为过渡时间和即将过渡到的场景。

Cocos2d-x 支持的场景过渡效果

　　Cocos2d-x 支持多种场景过渡效果，所有实现场景过渡效果的类都继承自TransitionScene，它们的create方法都有两个参数：第一个参数是场景过渡特效所用的时间；第二个参数是即将进入的场景。以下是部分过渡效果类及其描述：
　　★ TransitionRotoZoom：当前场景旋转变小，新场景旋转变大，实现转角切换效果。
　　★ TransitionJumpZoom：跳跃式，当前场景跳跃变小，新场景跳跃变大。
　　★ TransitionMoveInL：新场景从屏幕的左侧移入。
　　★ TransitionSlideInL：新场景从左侧滑入屏幕。
　　★ TransitionShrinkGrow：当前场景缩小，新场景在其之上变大，实现交错切换效果。
　　★ TransitionFlipX：水平翻转场景。
　　★ TransitionFlipAngular：一半水平翻转，一半垂直翻转场景。
　　★ TransitionZoomFlipX：水平翻转场景，同时有一定缩放效果。
　　★ TransitionZoomFlipAngular：一半水平、一半垂直翻转场景，同时有一定缩放效果。
　　★ TransitionFade：淡入淡出，原场景淡出，新场景淡入。
　　★ TransitionCrossFade：交叉淡入淡出效果。
　　★ TransitionTurnOffTiles：将当前场景随机分成方块，用分成方块的新场景随机地替换分成方块的旧场景。
　　★ ……
