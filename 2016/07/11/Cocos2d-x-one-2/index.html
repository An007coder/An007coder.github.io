Cocos2d-x的Layer、Director与Sprite的概述

发表于 2016-07-11   |   分类于 Cocos2d-x基础知识系列   |   | 阅读量 次

　　上一篇Blog写了Cocos2d-x核心类的Node与Scene，这篇Blog写Cocos2d-x核心类的Layer、Director与Sprite。

Layer 层类

　　Layer（层）与Scene（场景）类似，也有着容器的特性，与Scene不同的是Layer为屏幕上可绘制的区域，Layer通常包含的是直接呈现在屏幕上的具体内容，如Sprite（精灵）、Button（按钮）等游戏元素。
　　Layer类本身并不做什么，它的功能是允许触摸、加速度计和键盘输入等。因为大多数游戏都会接收基本的触摸输入，所以Layer通常是第一个被加入Scene的类。Cocos2d-x中每个Scene都包含一个或多个Layer，每一层负责各自的任务，每一层都有各自的游戏元素，彼此叠加。多个层组合在一起就构成了游戏显示的UI界面。
　　简而言之，Layer类直接继承Node类，作为精灵和其他游戏元素的容器，同时可以接收触摸、加速度计和键盘输入的信息等，前提是触摸、加速度计和键盘输入等功能已经开启。
　　为了方便游戏开发者，Cocos2d-x内置了几种特殊的Layer，具体如下：
　　♠ScrollView：常用于大尺寸的图片或大区域无限制地显示内容。子类TableView常用于大量数据的排列展示，特别是需要一些对齐格式的内容。这两个类常用来完成游戏菜单表格视图。
　　♠ LayerColor：一个单纯的实心色块。子类LayerGradient可以设置两种颜色的渐变效果。
　　♠ Menu：菜单。

Layer 类的常用成员变量

　　♦ bool _touchEnabled：是否支持触摸事件。
　　♦ bool _accelerometerEnabled：是否支持加速度计的事件。
　　♦ EventListener* _touchListener：触摸事件监听器。
　　♦ EventListenerAcceleration* _accelerationListener：加速度计的事件监听器。

Layer 类的常用函数

　　♥ virtual bool onTouchBegan(Touch* touch, Event* unused_event)：处理用户触摸时的按下事件，若需要继续处理该按下对应的后续事件，则返回true，否则返回false。返回false则onTouchMoved等函数不会被调用到。
　　♥ virtual void onTouchMoved(Touch* touch, Event* unused_event)：处理用户触摸时移动的事件。
　　♥ virtual void onTouchEnded(Touch* touch, Event* unused_event)：处理用户触摸时松开的事件。
　　♥ virtual void onTouchCancelled(Touch* touch, Event* unused_event)：处理用户触摸时被打断的事件。
　　♥ virtual void onTouchesBegan(const std::vector<Touch*>& touches, Event* unused_event)：处理用户的按下事件，多点触摸。
　　♥ virtual void onTouchesMoved(const std::vector<Touch*>& touches, Event* unused_event)：处理用户的移动事件，多点触摸。
　　♥ virtual void onTouchesEnded(const std::vector<Touch*>& touches, Event* unused_event)：处理用户的松开事件，多点触摸。
　　♥ virtual void onTouchesCancelled(const std::vector<Touch*>& touches, Event* unused_event)：处理用户被打断的事件，多点触摸。

Director 导演类

　　Director（导演类）是Cocos2d-x游戏引擎的核心，负责创建管理应用或者游戏的主窗口，并在特定的条件下执行某个场景。Cocos2d-x游戏是由不同的场景构成的，有Director负责运行和切换各个场景。在Cocos2d-x中，Director在任何一个时间点上只能运行一个场景。Director类是一个单例类，可以通过调用它的静态函数（getInstance()）来获取Director实例。
　　导演类的主要作用的如下：
　　♣ 初始化并设置OpenGL环境。
　　♣ 创建、访问和切换场景。
　　♣ 访问Cocos2d-x的配置细节。
　　♣ 访问视图（OpenGL ES、UIView和UIWindow）。
　　♣ 暂停、恢复和结束游戏。

Director 类的常用成员变量

　　♠ _runningScene：当前运行的场景，Director一次只能显示一个场景，变量类型为Scene。
　　♠ _animationInterval：游戏的FPS（帧每秒）值，变量类型为double。
　　♠ _openGLView：OpenGl视图，所有要进行视觉呈现的对象都会在该视图中渲染，变量类型为EGLView。
　　♠ _paused：Director是否被暂停，变量类型为bool。
　　♠ _totalFrames：自Director对象启动以来已调用帧的个数。
　　♠ _sendCleanupToScene：判断被替换的场景是否收到cleanup（清除）消息。如果新场景时弹出的，则原来的场景不会收到清除消息；如果新场景直接替换旧场景，则原来的场景会收到清除消息，变量类型为bool。

Director 类的常用函数

　　♥ static Director* getInstance()：返回一个导演类的单例实例。
　　　auto director = Director::getInstance();
　　♥ const Size& getWinSize() const：返回OpenGL ES的视图大小。
　　　Size size = Director::getInstance()->getWinSize();
　　♥ Size getVisibleSize() const：返回可看见模拟器的OpenGL ES的视图尺寸大小，返回值类型为Size，如果没有调用EGLView::setDesignResolutionSize()函数的话，返回的值与getWinSize()函数一样。
　　　Size visibleSize = Director::getInstance()->getVisibleSize();
　　♥ void runWithScene(Scene* scene)：进入导演的主循环并启动给定的场景，调用该函数运行第一个场景，运行游戏的第一个场景只能用这个函数，而且这个函数只能运行第一个场景，不能运行其他场景。
　　　Director::getInstance()->runWithScene(scene);
　　♥ void pushScene(Scene* scene)：暂停当前运行的场景，并将其压栈到场景堆栈中，将新传入的场景设置为当前场景。
　　　Director::getInstance()->pushScene(scene);
　　♥ void popScene()：从场景堆栈中弹出栈顶的场景作为当前场景。
　　　Director::getInstance()->popScene();
　　♥ void popToRootScene()：从场景堆栈中弹出所有场景，直接返回运行主场景。
　　　Director::getInstance()->popToRootScene();
　　♥ void replaceScene(Scene* scene)：直接使用传入的新场景替换当前场景，并释放当前场景。
　　　Director::getInstance()->replaceScene(scene);
　　♥ void end()：结束当前运行中的场景，并释放该场景。
　　　Director::getInstance()->end();
　　♥ void pause()：暂停当前运行中的场景，调用该函数后当前画面还会存在，但预定的更新方法会停止。
　　　Director::getInstance()->pause();

Sprite 精灵类

　　精灵（Sprite）是游戏开发中的一个术语。精灵是一个图像，可以是任何一个游戏元素。一般情况下，精灵来自PNG或者PVRTC图像。当图像载入内存后，就会被转成纹理图像，在设备或模拟器的屏幕上被渲染。Sprite类继承自Node类，代表了游戏中最小的可见单位，由于精灵装载了一个平面纹理，具有丰富的表现力，而且可以通过多种方式加载，所以可以说精灵是游戏中最灵活的。

纹理的相关概念

　　Cocos2d-x中几乎所有游戏角色都是以PNG或JPEG格式的图像存储在设备的内存中，这些图象一旦被加载入内存，它们将以一种未压缩的纹理格式来存储。推荐使用PNG格式的图像存储。在游戏运行中，所有的图像文件都会被加载为GPU可以解释的OpenGL ES纹理图像，而所有的精灵则对应着这些纹理图像。Cocos2d-x中内置了一个纹理缓存区（TextureCache）来保存这些纹理图，当创建多个相同的精灵对象时，系统会充分地利用已缓存的纹理图像，避免每次都去创建一个纹理图像，从而提高程序的运行效率。

纹理（Texture）

　　可以认为纹理就是一张图片，这张图片被加载进内存，由精灵显示出来。更深层地讲，纹理是3D游戏中绘制到物体表面上的图案。图像文件在使用前必须转换成设备可以解释的格式，并加载到内存中。但是有一点需要注意，纹理的宽度与高度必须是2的n次方，如1024×512或128×64等。当图像不符合这个要求时，系统会自动调整纹理的尺寸，所以纹理会比原图像占用更多的内存。

纹理图集（TextureAtlas）

　　因为纹理图的缘故，每次通过图片创建精灵都需要加载纹理图，为了提高程序的运行效率，通常会把这些纹理图拼合为一个纹理图，拼合后的纹理图称为纹理图集（TextureAtlas）。事实上，纹理图集就是一张尺寸为2的n次方的大图像而已，其中包含了许许多多的小图像。里面的每张小图像都有与之对应的一张小图像的矩形，Cocos2d-x可以从纹理图集中取出特定的某个图像进行渲染。纹理图集的信息通常保存在独立的.plist文件中。使用精灵表单类（SpriteBatchNode）可以将所有的图像一次性交给OpenGL ES来处理，这样比把单个图像逐个交给OpenGL ES来处理要高效得多。

纹理相关的类

　　♣ Texture2D（纹理）：从图片、文本或源数据中创建OpenGL 2D纹理图像，所创建的纹理图像使用2的n次方来填充。
　　♣ TextureCache（纹理缓存）：该类是单例类，用于加载和管理纹理图像。一旦纹理图像成功加载完成后，下次使用时可使用它返回之前加载的纹理图像，从而减少对GPU和CPU内存的占用，提高效率。
　　♣ TextureAtlas（纹理图集）：纹理图集文件可以是任何一种Texture2D（纹理）所支持的文件类型。并且可以对纹理图集的矩形进行实时的更新、添加、删除或重新排序操作。

Sprite 精灵类的常用成员变量

　　Sprite类继承自Node类，除了从Node类继承的变量外，还有以下的成员变量：
　　♦ _textureAtlas：仅在该精灵使用SpriteBatchNode渲染时可用，类型为TextureAtlas，改变为TextureAtlas的引用
　　♦ _atlasIndex：精灵表单的索引，变量类型为int，不建议修改该值。
　　♦ _batchNode：对SpriteBatchNode的引用，变量类型为SpriteBatchNode。
　　♦ _dirty：精灵是否需要在纹理集中更新，变量类型为bool。
　　♦ _blendFunc：遵循TextureProtocol协议，变量类型为BlendFunc（纹理的混合使用）。
　　♦ _texture：Texture2D对象，用于获取或设置精灵所用的纹理。
　　♦ _rect：Texture2D对象的矩形，变量类型为Rect。
　　♦ _rectRotated：纹理矩形是否被旋转，变量类型为bool。
　　♦ _offsetPosition：以点值计量的精灵位置的偏移，变量类型为Point。
　　♦ _quad：顶点坐标、纹理坐标和颜色信息，变量类型为V3F_C4B_T2F_Quad。
　　♦ _flippedX：精灵是否沿着水平方向翻转，变量类型为bool。
　　♦ _flippedY：精灵是否沿着垂直方向翻转，变量类型为bool。

Sprite 精灵类常用的函数

　　除了从Node类继承的函数外，还有以下常用函数：
　　♠ static Sprite* create()：静态函数，创建一个空的精灵对象，之后可以调用setTexture函数设置纹理图像。
　　♠ static Sprite* create(const std::string& filename)：静态函数，根据一个图像文件名称创建一个精灵。
　　♠ static Sprite* create(const std::string& filename, const Rect& rect)：静态函数，根据图像文件名创建一个矩形为rect的精灵。
　　♠ static Sprite* createWithTexture(Texture2D *texture)：静态函数，使用纹理图像创建一个精灵，默认位置偏移为(0,0)。
　　♠ static Sprite* createWithTexture(Texture2D *texture, const Rect& rect, bool rotated=false)：静态函数，使用纹理图像创建一个矩形为rect的精灵，位置偏移为(0,0)。
　　♠ static Sprite* createWithSpriteFrame(SpriteFrame *pSpriteFrame)：静态函数，使用精灵帧创建一个精灵。
　　♠ static Sprite* createWithSpriteFrameName(const std::string& spriteFrameName)：静态函数，使用精灵帧的名称创建一个精灵。该函数在使用时，将从SpriteFrameCache中获取SpriteFrame，如果精灵帧缓存中不存在该精灵帧，系统将抛出一个异常。
　　♠ virtual bool initWithTexture(Texture2D *texture)：使用纹理图像初始化一个精灵，默认偏移位置为(0,0)，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithTexture(Texture2D *texture, const Rect& rect)：使用纹理图像初始化一个矩形为rect的精灵，位置偏移是(0,0)，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithTexture(Texture2D *texture, const Rect& rect, bool rotated)：使用纹理图像初始化一个矩形为rect的精灵，可选择是否旋转，位置偏移是(0,0)，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithSpriteFrame(SpriteFrame *pSpriteFrame)：使用精灵帧初始化一个精灵，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithSpriteFrameName(const std::string& spriteFrameName)：使用精灵帧的名称初始化一个精灵，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithFile(const std::string& filename)：根据一个图像文件名称初始化一个精灵，初始化成功返回true，否则返回false。
　　♠ virtual bool initWithFile(const std::string& filename, const Rect& rect)：根据一个图像文件名称初始化一个矩形为rect的精灵，初始化成功返回true，否则返回false。
　　♠ virtual void updateTransform(void)：根据旋转角度、位置和比例值更新精灵所在的矩形。
　　♠ virtual SpriteBatchNode* getBatchNode(void)：返回精灵表单对象。
　　♠ virtual void setBatchNode(SpriteBatchNode *spriteBatchNode)：设置精灵表单对象。
　　♠ virtual void setTextureRect(const Rect& rect)：设置纹理图像的矩形大小。
　　♠ virtual void setSpriteFrame(SpriteFrame *newFrame)：为Sprite设置新的显示帧。
　　♠ virtual SpriteFrame* getSpriteFrame() const：返回当前精灵显示帧。
　　♠ virtual void setDisplayFrameWithAnimationName(const std::string& animationName, int frameIndex)：根据动画名称和索引更改要显示的帧，动画名将从AnimationCache中获取。
　　♠ bool isFlippedX(void) const：精灵是否沿水平方向翻转。
　　♠ void setFlippedX(bool flippedX)：设置精灵是否沿水平方向翻转。
　　♠ bool isFlippedY(void) const：精灵是否沿垂直方向翻转。
　　♠ void setFlippedY(bool flippedY)：设置精灵是否沿垂直方向翻转。

SpriteFrame 精灵帧类

　　SpriteFrame（精灵帧）是相对于动画动作而产生的概念。一个精灵是固定的节点，它可以拥有许多精灵帧，通过快速地更换精灵帧来实现动态效果。与之前介绍的节点类不同，SpriteFrame类不是继承自Node类，而是直接继承自Ref类，Node类继承自Ref类。SpriteFrame用来表示一个精灵，包括Texture2D纹理和矩形大小。

　　SpriteFrame的常用成员变量如下：
　　♥ _rect 和 _rectInPixels：表示精灵帧的矩形大小，分别用point（点）值和pixels（像素）值来表示，变量类型为Rect。它们是彼此相连的，任一成员发生变化都会自动更新另一个成员。
　　♥ _offset 和 _offsetInPixels：精灵帧的位置偏移，分别用point（点）值和pixels（像素）值来计算，变量类型为Point。
　　♥ _originalSize 和 _originalSizeInPixels：修剪后图片的原始大小，分别用point（点）值和pixels（像素）值来计算，变量类型为Size。
　　♥ _texture：精灵帧的纹理，变量类型为Texture2D。
　　♥ _textureFilename：精灵帧的纹理文件名称，变量类型为std::string。

　　SpriteFrame的常用函数如下：
　　♣ static SpriteFrame* create(const std::string& filename, const Rect& rect)：使用纹理图名称和矩形（以点值表示）创建一个SpriteFrame。
　　♣ static SpriteFrame* create(const std::string& filename, const Rect& rect, bool rotated, const Point& offset, const Size& originalSize)：使用纹理图名称、矩形（以像素值表示）、是否旋转、位置偏移和原始大小等创建一个SpriteFrame。
　　♣ static SpriteFrame* createWithTexture(Texture2D* pobTexture, const Rect& rect)：使用纹理图和矩形（以点值表示）创建一个SpriteFrame。
　　♣ static SpriteFrame* createWithTexture(Texture2D* pobTexture, const Rect& rect, bool rotated, const Point& offset, const Size& originalSize)：使用纹理图、矩形（以像素值表示）、是否旋转、位置偏移和原始大小等创建一个SpriteFrame。

SpriteFrameCache 精灵帧缓存类

　　SpriteFrameCache（精灵帧缓存类）直接继承自Ref类，是一个单例类，所有精灵可以共享，它的主要作用是存储精灵帧来提高程序的运行效率。它没有提供特别的成员变量，而是提供了一系列用于管理SpriteFrame的函数，常用函数如下：
　　♦ static SpriteFrameCache* getInstance(void)：获取精灵帧缓存类的单例对象。
　　auto frameCache = SpriteFrameCache::getInstance();
　　♦ static void destroyInstance()：清除SpriteFrameCache中的缓存，删除所有精灵帧，并释放所保留的变量。
　　SpriteFrameCache::destroyInstance();
　　♦ void addSpriteFramesWithFile(const std::string& plist)：从plist属性文件中添加多个精灵帧。纹理将会被自动载入，纹理的扩展名使用.png替换.plist。
　　SpriteFrameCache::getInstance()->addSpriteFramesWithFile("resource.plist");
　　♦ void removeSpriteFrames()：清除已加载的精灵帧的词典。
　　SpriteFrameCache::getInstance()->removeSpriteFrames();
　　♦ void removeSpriteFrameByName(const std::string& name)：根据名称从精灵帧缓存中删除一个精灵。
　　SpriteFrameCache::getInstance()->removeSpriteFrameByName("player.png");
　　♦ void removeUnusedSpriteFrames(void)：删除所有未使用的精灵帧，所有retain计数为1的精灵帧都会被删除。当游戏切换新场景时可以调用此方法。
　　SpriteFrameCache::getInstance()->removeUnusedSpriteFrames();
　　♦ void removeSpriteFramesFromFile(const std::string& plist)：从plist属性文件中删除多个精灵帧。存储在该plist文件中的精灵帧都会被删除。
　　SpriteFrameCache::getInstance()->removeSpriteFramesFromFile("resource.plist");
